  [DONE]  1.1  Lexer
               Target: compiler/src/main/kotlin/nv/compiler/lexer/

               - Token.kt         — data class Token(kind, text, span)
               - LexerError.kt    — sealed error hierarchy with source locations
               - Lexer.kt         — hand-written lexer:
                   * NFC-normalises source via java.text.Normalizer before scanning
                   * All keywords, Unicode operators (∀ ∃ ∑ ∏ ∈ ∧ ∨ ¬ → ≤ ≥ ≠ ÷ ⊕ π ∞)
                   * INDENT/DEDENT injection (tab=4 spaces; mixing → MixedIndentation error)
                   * NEWLINE suppression inside open ()[]{}
                   * String interpolation mode stack:
                       STR_START / STR_TEXT / INTERP_START / <inner tokens> / INTERP_END / STR_END
                   * Raw strings: r"..." and r"""..."""
                   * Maximal munch for |>, ->, ?., ??, <<=, >>=, compound assignments
                   * // is always a comment (no integer-division // token; ÷ is INT_DIV)
               - tests/src/test/kotlin/nv/tests/LexerTest.kt — 126 tests, all passing

  [DONE]  1.5  IR lowering (HIR → MIR → LLVM IR)
               Target: compiler/src/main/kotlin/nv/compiler/codegen/
               - CodegenError.kt  — sealed error hierarchy (UnsupportedFeature, InternalError)
                                    each with message: String and span: SourceSpan?
               - CodegenResult.kt — Success(llvmIr: String) / Failure(errors) sealed result
               - LlvmIrEmitter.kt — ~1850-line LLVM IR emitter:
                   * Typed-pointer IR style (i8*, i8**, $lt*) — compatible with Apple Clang 12+
                   * Four StringBuilder sections: globals, declares, rtFns, userFns
                   * Per-function alloca-based locals (alloca/load/store for all vars)
                   * llvmType(): TInt/TInt64→i64, TFloat/TFloat64→double, TBool→i1,
                     TStr→i8*, TUnit/TNever→void
                   * stringConst(): string interning with GEP-based i8* extraction
                   * Runtime functions inlined per module: @nv_print, @nv_println,
                     @nv_print_int, @nv_println_int, @nv_print_float, @nv_println_float,
                     @nv_print_bool, @nv_println_bool, @nv_int_to_str, @nv_float_to_str,
                     @nv_bool_to_str, @nv_str_concat, @nv_str_eq, @nv_str_len, @nv_panic
                   * Control flow: if/else, while, for-in (array), match (equality chains),
                     short-circuit &&/||, inline if expressions (phi nodes)
                   * String interpolation desugared to @nv_str_concat chains
                   * Power operator ^ → @pow call; ÷ → sdiv; ^ float → @pow
                   * main() lowered to define i32 @main(i32 %argc, i8** %argv)
                   * User functions mangled to @nv_<name>
                   * isTerminated flag prevents unreachable instructions after ret/br
                   * loopStack for break/continue (while.end / while.cond targets)
               - Compiler.kt updated: LlvmIrEmitter wired after TypeChecker;
                 CompileResult.IrSuccess(llvmIr) added to sealed result class
               - tests/src/test/kotlin/nv/tests/CodegenTest.kt — 27 tests:
                   * IrStructure (21): check IR string content (no clang needed)
